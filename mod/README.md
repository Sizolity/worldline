# Worldline Mods

`mod/` 是 Worldline 的内容目录。所有 GM 风格、剧情世界、人物设定都以 markdown
文件的形式住在这里，**你随时可以打开任何一个 `.md` 编辑**——这就是 mod。

```
mod/
├── styles/                          GM 风格：怎么说话、什么语气、什么语言
│   └── shuoshu/                       v1 唯一示例风格（白话评话/说书味）
└── scenarios/                       剧情世界：什么角色、什么规则、什么线索
    └── xiyou-changan/                 西游记 · 长安启程（v1 示例场景）
```

> v1 阶段只有 `shuoshu` 一种 GM 风格、`xiyou-changan` 一个场景作为示例。其它风格 /
> 场景由作者按本文档约定自行扩展。引擎不再内置任何 fallback 风格，所有 prompt
> 内容都从 mod 加载——`--scenario` / `--style` 必须显式指定。

## 一句话上手

```bash
# 用 shuoshu 风格玩西游剧情
rpg-cli seed --workspace /tmp/wl --scenario xiyou-changan --style shuoshu
rpg-cli play --workspace /tmp/wl --world-id xiyou-changan

# 自己拷一份 shuoshu 改风格后，用新风格玩同一个剧情（剧情不变，旁白的嘴换了）
cp -r mod/styles/shuoshu mod/styles/my-style
$EDITOR mod/styles/my-style/persona/narrator.md
rpg-cli play --workspace /tmp/wl --world-id xiyou-changan --style my-style
```

一局游戏 = **一个 style** × **一个 scenario** × **一个 locale**。

## 机制定位（v1）

Worldline **不是** TRPG，**也不是**带战斗系统的 CRPG。它是一个**纯叙事的 LLM-GM
游戏**——LLM 全权负责讲故事、控 NPC、推进世界，**绝不算数**。

### v1 的三条产品红线

1. **零数值系统**——角色没有 HP / 攻防 / 等级 / 属性表。能力强弱是**叙事知识**
   （"齐天大圣 vs 一个凡间和尚"），不是数字
2. **零可见判定**——没有"掷骰中… 14! 失败"这种 UI。骰子是引擎内部的随机性工具，
   玩家**永远只看到故事**，看不到机制
3. **零规则书**——没有 d20 / d6 / DC / AC。CRPG/JRPG 的**惊喜感**（这次爆击、那次失手）
   由 LLM 隐形使用随机性来实现，作者写 mod 时**完全不接触数值**

### 这意味着

- 写 `characters/<name>.md` 时只写**是谁**（身份、性格、外貌、口吻、关键能力的叙事描述），
  **不写数值**
- 写 `rules.md` 时只写**世界观规则**（叙述风格、因果、紧箍咒怎么生效），
  **不写机制公式**
- 写 `threads.md` 时只写**剧情大纲**（标题 + 简介），**不写难度 / 张力数字**

> **机制层（隐藏世界线）现在是可选的「已实现」功能**：在 scenario 里加一个
> `worldlines/` 目录即可让某条 thread 拥有"暗中演化"的能力（随时间漂移、到点触发
> 后果）。这一层**完全 opt-in**——没有 `worldlines/` 目录的 scenario 行为和以前
> 完全一样。它仍然恪守上面的**零数值红线**：作者只写"词"（节奏 / 张力档位 / 状态
> 词），所有数字都藏在引擎的关键词表里，作者永远不接触数值。详见下方
> 「worldlines/ ——隐藏世界线（可选机制层）」。

## styles/ ——GM 风格包

定义 GM（旁白 / 知识员 / 行动建议器）怎么讲故事。**不含任何具体世界内容**。
切换 style，故事不变，只是讲述方式换了一个人。

```
styles/<name>/
├── persona/                         GM 角色的设定
│   ├── narrator.md                  旁白（必须有）
│   ├── lorekeeper.md                世界编年史记录员
│   └── action_suggester.md          行动建议器
└── scene/                           场景脚本
    ├── prologue.md                  新开局的开场白
    └── recap.md                     续玩时的接续场景
```

### 做一个新 style

```bash
cp -r mod/styles/shuoshu mod/styles/my-style
$EDITOR mod/styles/my-style/persona/narrator.md
rpg-cli play --workspace ... --world-id ... --style my-style
```

## scenarios/ ——剧情世界包

定义一个具体的故事世界。哪些角色、什么规则、什么线索、初始状态如何。
**和 GM 风格无关**——同一个剧情可以配不同 style。

```
scenarios/<name>/
├── world.md                         世界观（标题、基调、类型、开场背景） + start_location
├── rules.md                         世界观规则（叙述风格、因果、特殊设定）
├── threads.md                       剧情线索
├── characters/                      角色，一个角色一个文件
│   ├── sun_wukong.md                  必有一个 role=player 的玩家角色
│   └── ...
├── locations/                       地点，一个地点一个文件
│   └── ...
├── events/                          初始事件（启程描述等）
│   └── ...
└── worldlines/                      （可选）隐藏世界线 / 暗中演化的机制层
    └── ...
```

详见 `xiyou-changan/` 下的文件结构。

### scenario 的"配置面"= world.md 一个字段

scenario 包里**没有**单独的配置文件。所有结构化字段要么由约定推导，要么压缩到
`world.md` 顶部的 frontmatter：

| 信息 | 怎么来 |
|---|---|
| scenario ID | 目录名（`scenarios/xiyou-changan/` → `xiyou-changan`） |
| 标题 | `world.md` 的 H1 |
| 简介 / 类型 / 基调 | `world.md` 正文 |
| 玩家角色 | 扫 `characters/*.md`，找 `role: player` 那个（必须有且只有一个） |
| 起始地点 | `world.md` frontmatter 的 `start_location`（指向 `locations/<name>.md` 文件名） |
| GM 风格 | **不在 scenario 里**——play 时通过 `--style` 选，必填 |
| locale | v1 阶段统一 zh-CN |

```markdown
---
start_location: changan_gate
---

# 西游记 · 长安启程

贞观十三年秋……
```

作者面对的就这一个 frontmatter 字段。其它想推荐什么（"建议配 seanchaidh 风
食用"）直接写在正文段，给玩家看就完了。

## Markdown 约定

### 角色 / 地点 / 事件 / 物品（一条记录一个文件）

正文写描述，**不写数值、不写状态字段**。引擎从 H1 取名字、从文件名派生 ID、
其它字段全部走默认值。

唯一的 frontmatter 字段是 `role`，用来区分**玩家角色**和 NPC：

```markdown
---
role: player
---

# 孙悟空

齐天大圣，七十二变……
```

`role` 取值：

- `player`——玩家操控的角色（一个 scenario 必有且只有一个）
- `npc`——非玩家角色（默认，可省略）

location / event 文件**无需 frontmatter**——直接 H1 + 正文即可。

> v1 阶段角色无属性表（HP / 攻防 / 等级…），详见上方"机制定位（v1）"段。

### threads / rules（一个类别一个文件）

threads / rules 这类稀疏数据写成"大纲式"——每条记录一个 **H2 段**：

- H2 标题 = 这条线索 / 规则的人类可读名字
- H2 下面的散文 = 这条线索的简介、或规则的正文

```markdown
## 西天取经

三藏师徒奉旨西行求取大乘真经，途经十万八千里，九九八十一难。

## 师徒嫌隙

悟空性烈，三藏慈悲，紧箍咒一念一痛。
```

引擎做的事：

- 自动从 H2 标题派生唯一 ID（同名时附序号）
- 其它结构化字段（类型 / 状态 / 张力 / 分类等）**全部走默认值**，作者不用关心
- 想真的需要把某条线索标成"伏笔 / 主线"、给某条规则打分类标签，**v1 阶段不需要**——
  默认值已经够用，未来再开放轻量的标记机制

facts / relations 在 v1 阶段不参与默认 scenario（xiyou-changan 用不到），
等具体场景需要时再讨论它们的最小写法。

### worldlines/ ——隐藏世界线（可选机制层）

`worldlines/` 是一个**可选**目录。没有它，scenario 照常运行；有了它，你可以让某条
thread 在背后"自己演化"——随时间累积张力，到达某个档位时触发后果（改 NPC 态度、把
线索摆上台面等）。**玩家看不见这条线**，只会从旁白里感到气氛在变。

一条隐藏世界线 = `worldlines/` 下的一个 `.md` 文件，例如
`worldlines/shitu.md`（线索 ID 自动派生为 `wl_shitu`）：

```markdown
---
thread: 师徒嫌隙        # 必填：指向 threads.md 里某个 H2 标题
visibility: hidden      # 可选：默认 hidden
stage: 初行             # 可选：起始阶段名
tempo: 渐磨             # 可选：漂移节奏（见下表），省略=不漂移
---

# 师徒嫌隙

（这里随便写风味散文，引擎忽略，只给作者自己看。）

## 微疑

触发：师徒张力·初现
结果：
- 唐三藏：微疑

## 决裂

触发：师徒张力·决裂
结果：
- 师徒嫌隙：浮上台面
- 唐三藏：心生芥蒂
```

读法：

- **frontmatter `thread`**——填 threads.md 里的 H2 **标题**（人类可读名），引擎自动
  解析成对应的线索。**绝不要写 `thread-2` 这种内部 ID。**
- **每个 H2 = 一个里程碑**，按出现顺序自动编号（`m1`、`m2`…）。H2 标题只是给你看的
  注释，不影响逻辑。
- **`触发：…·<档位>`**——`·` 前面是随便写的描述（"师徒张力"），引擎只认最后那个**张力
  档位**关键词。默认就盯 frontmatter 绑定的那条 thread，不用重复写线索名。
- **`结果：` 下面的 `- 目标：词` 列表**——
  - 目标若是某条 thread 标题（如"师徒嫌隙"）→ 改这条线索的**状态**（词见状态表）；
  - 否则按**角色名**（character 文件的 H1）解析 → 给这个 NPC 写一条**态度**（`词`）。
- 名字解析**失败即报错**（找不到 / 同名歧义都会在 seed 时直接报错），不会悄悄忽略。

**作者只写"词"，不写数字。** 节奏 / 张力档位 / 状态词对应的具体数值与引擎常量都由
引擎的关键词表负责（守住零数值红线）。v1 内置词表如下：

| 节奏 `tempo`（漂移快慢） | 张力档位（`触发` 用） | 状态词（目标是 thread 时） |
|---|---|---|
| `微澜` 极缓 | `初现` | `浮上台面` → active |
| `渐磨` 渐进（示例用） | `加深` | `开启` → open |
| `紧迫` 急促 | `决裂` | `潜伏` → dormant |
|  | `临界` | `了结` → resolved / `破裂` → failed / `搁置` → abandoned |

> 想扩充词表（加新节奏 / 新档位 / 新状态词），v1 阶段改引擎里的关键词表即可；未来会
> 考虑开放 `mechanics/` 覆盖文件，但当前不需要。

### Mod 的编辑边界——三层架构

为了既给作者自由创作的空间，又守住游戏整体的输出稳定性，mod 内容按下面三层划分：

#### 1. 你的自由编辑区

mod 作者**完全可写、完全可改**的部分。

| 类别 | 例子 | 写在哪 |
|---|---|---|
| GM 身份 | "你是 RPG 的旁白" / "说书人" / "守密人" | persona 文件正文段（H1 + 第一段） |
| 写作风格 | "节制、留白" / "感官细节优先" | `## 风格` 段 |
| 世界风味契合 | "措辞贴合 genre" / "NPC 对白按身份" | `## 风格` 段 |
| 输出节奏 | "每回合 2-4 段" | `## 风格` 段（**有上限，超过引擎会警告**） |
| 自定义提示 | `## 注意事项` / `## 工具说明` 等任何**非保留**的 H2 | 自由命名 |

#### 2. 世界状态占位区（保留 H2 标题）

下列 H2 标题在 persona 文件里**是位置占位符**——你写在哪里，引擎就在哪里注入当前世界状态。
**你控制位置，引擎控制内容**：

| 保留 H2 | 引擎填什么 | 何时填 |
|---|---|---|
| `## 世界` | 当前世界名 / 类型 / 基调 | 总是 |
| `## 规则` | 启用中的世界观规则（来自 scenario 的 rules.md） | 总是 |
| `## 角色` | 在场 character 类实体 | 总是 |
| `## 地点` | 在场 location 类实体 | 总是 |
| `## NPC 记忆` | 各 NPC 累积的长期 / 短期记忆 | 总是 |
| `## 最近事件` | 最近几条 world event | 总是 |
| `## 当前线索` | 活跃的 story thread | 总是 |
| `## 发现协议` | fog 探索协议（语义、状态、机制由引擎管理） | 仅 fog 启用；关闭时整段移除 |

> 你写这些 H2 时**不要在下面填内容**——填了也会被引擎替换掉。引擎只替换 H2 下到
> 下一个 H2 之间的范围。

#### 3. 引擎合规层——你看不到、动不了

下列内容**完全由引擎管理**，**不出现在任何 mod 文件里**，也无法编辑或调位置。
它们在渲染最终 system prompt 时由引擎追加到末尾：

| 内容 | 为什么不给用户改 |
|---|---|
| 工具名 + 使用指引 | mod 写错工具名会让模型幻觉；工具集随 disclosure 动态变 |
| 反幻觉约束 | "不要编造世界里不存在的实体" 是世界真实性硬保证，关掉就乱套 |
| 跨系统协同规则 | "结尾不许罗列玩家选项" 等是和其它模块（选项 UI）的契约 |
| 玩家代理性原则 | "不替玩家做选择" 是产品级硬底线 |
| Fog 状态机语义 | known/explored/hidden 三态的具体行为 |
| 工具调用披露纪律 | "不要写'调用 roll 工具'这种元话语" |

简单记：**你写"GM 是谁、怎么讲故事、什么世界风味"，引擎负责"系统底线、机制对接、安全约束"**。
两边各管各的，互不串台。

### 不是保留的 H2 都是你的

`## 风格`、`## 注意事项`、`## 我的特殊设定` 这种**不在第 2/3 类保留名单**的 H2，
引擎一字不动，完整保留你写的内容。想加什么自定义段，写就完了。

## 加载路径

引擎按下面三个位置依次找 `mod/` 目录：

1. `WORLDLINE_MOD_DIR` 环境变量
2. 当前工作目录下的 `mod/`
3. 可执行文件所在目录下的 `mod/`

三个都找不到则直接报错退出。**正常发布产物里 `mod/` 和 `rpg-cli` 在同一目录**，
所以普通用户不需要管这个。
